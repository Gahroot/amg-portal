# Refresh Token Rotation with Revocation

## Overview
Implement refresh token rotation with family-based revocation to prevent stolen token reuse. When a refresh token is used, the old one is invalidated and a new one is issued. If a previously-used token is replayed (reuse detection), the entire token family is revoked.

Add a `hash_token` function using SHA-256 (store hashes, not raw tokens):

```python
def hash_token(token: str) -> str:
    """SHA-256 hash of a token for safe database storage."""
    return hashlib.sha256(token.encode("utf-8")).hexdigest()
```

Modify `create_refresh_token` signature to accept optional `family` and `jti` kwargs:

```python
def create_refresh_token(data: dict[str, Any], *, family: str | None = None, jti: str | None = None) -> str:
    to_encode = data.copy()
    expire = datetime.now(UTC) + timedelta(days=settings.REFRESH_TOKEN_EXPIRE_DAYS)
    token_jti = jti or str(uuid_mod.uuid4())
    token_family = family or str(uuid_mod.uuid4())
    to_encode.update({
        "exp": expire,
        "type": "refresh",
        "jti": token_jti,
        "family": token_family,
    })
    algorithm = _validate_algorithm()
    return jwt.encode(to_encode, settings.SECRET_KEY, algorithm=algorithm)
```

### 2. `backend/app/models/refresh_token.py` — Database model

Follow existing patterns from `device_session.py`:

```python
"""Refresh token storage for rotation and revocation."""

import uuid
from datetime import datetime

from sqlalchemy import Boolean, DateTime, ForeignKey, Index, String
from sqlalchemy.dialects.postgresql import UUID
from sqlalchemy.orm import Mapped, mapped_column

from app.db.base import Base, TimestampMixin


class RefreshToken(Base, TimestampMixin):
    """Stores refresh token hashes for rotation and reuse detection."""

    __tablename__ = "refresh_tokens"

    id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    user_id: Mapped[uuid.UUID] = mapped_column(
        UUID(as_uuid=True), ForeignKey("users.id", ondelete="CASCADE"), nullable=False, index=True,
    )
    token_hash: Mapped[str] = mapped_column(String(64), nullable=False, unique=True, index=True)
    jti: Mapped[str] = mapped_column(String(36), nullable=False, unique=True, index=True)
    family_id: Mapped[str] = mapped_column(String(36), nullable=False, index=True)
    is_revoked: Mapped[bool] = mapped_column(Boolean, nullable=False, default=False)
    expires_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), nullable=False)

    __table_args__ = (
        Index("ix_refresh_tokens_family_revoked", "family_id", "is_revoked"),
    )

    def __repr__(self) -> str:
        return f"<RefreshToken(id={self.id}, user_id={self.user_id}, revoked={self.is_revoked})>"
```

#### b) New helper function `_issue_refresh_token` (after `_clear_auth_cookies`)

```python
async def _issue_refresh_token(
    db: AsyncSession,
    user_id: str,
    token_data: dict[str, Any],
    *,
    family_id: str | None = None,
) -> str:
    """Create a refresh token, store its hash, return the raw JWT."""
    fid = family_id or str(uuid.uuid4())
    jti = str(uuid.uuid4())
    token = create_refresh_token(token_data, family=fid, jti=jti)
    expires_at = datetime.now(UTC) + timedelta(days=settings.REFRESH_TOKEN_EXPIRE_DAYS)
    db.add(RefreshToken(
        user_id=user_id,
        token_hash=hash_token(token),
        jti=jti,
        family_id=fid,
        is_revoked=False,
        expires_at=expires_at,
    ))
    return token
```

```python
async def _cleanup_expired_refresh_tokens_job() -> None:
    """Delete expired refresh tokens to keep the table lean."""
    try:
        from app.models.refresh_token import RefreshToken
        async with AsyncSessionLocal() as db:
            result = await db.execute(
                delete(RefreshToken).where(RefreshToken.expires_at < datetime.now(UTC))
            )
            await db.commit()
            count = result.rowcount
        if count:
            logger.info("Cleaned up %d expired refresh tokens", count)
    except Exception:
        logger.exception("Error cleaning up expired refresh tokens")
```

Register in `start_scheduler()` before `_scheduler.start()`:

## Commit Ordering Concerns

For login paths (lines 166, 209), the current code does `await db.commit()` to save `user.last_login_at` before creating tokens. After adding `_issue_refresh_token`, we need the commit to happen AFTER the token is stored. Solution: remove the early commit and let it happen after token storage, or add a second commit after. Safest: combine into one commit after all DB writes.

## Risks & Edge Cases

1. **Backward compatibility**: Existing refresh tokens won't have `jti`/`family` claims. They'll be rejected with 401 (users re-login). This is acceptable for a security upgrade.
2. **Race conditions**: Two simultaneous refreshes with the same token — one succeeds, the other triggers reuse detection. Acceptable.
3. **Table growth**: Mitigated by daily cleanup at 3 AM UTC.
